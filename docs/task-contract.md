# Task Contract

Jedes umsetzungsnahe Bead sollte mindestens diese Punkte klaeren:

## Problem

Welches konkrete Verhalten oder welche Luecke wird adressiert?

## Ziel

Was ist nach Abschluss sichtbar besser oder moeglich?

## Scope

Welche Schichten, Module oder Dateien duerfen veraendert werden?

## Nicht im Scope

Welche angrenzenden Themen werden bewusst nicht mit erledigt?

## Akzeptanzkriterien

- fachlich pruefbar
- technisch pruefbar
- klein genug fuer einen ueberschaubaren Inkrement

## Verifikation

Welche Tests, Checks oder manuellen Pruefschritte muessen laufen?

## Folgefragen

Welche Anschlussarbeit ist wahrscheinlich, aber nicht Teil dieses Beads?
