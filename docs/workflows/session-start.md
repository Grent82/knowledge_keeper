# Session Start Workflow

## Pflichtschritte

```bash
bd prime
bd ready
bd show <id>
bd update <id> --claim
```

## Danach

1. betroffene Schichten benennen
2. relevante Dokumente lesen
3. Implementations- oder Verifikationsplan knapp festhalten
4. erst dann Dateien aendern

## Wenn kein passendes Bead existiert

- neues Bead anlegen
- Ziel und Akzeptanz dokumentieren
- dann claimen
